# AI-Powered Lung Cancer Diagnosis Assistant

Final-year project. Detects lung cancer (Normal / Benign / Malignant) from CT
scans using deep learning, with clinical risk prediction, explainability, and
a RAG-based awareness chatbot planned as later modules.

## Current status

**Built so far: Module 1 — Medical Image Classification.**
Everything else in the full spec (clinical risk model, Grad-CAM, LLM report
generation, RAG chatbot, doctor dashboard, FastAPI backend, database,
deployment) is scaffolded as empty folders and will be filled in module by
module — see `docs/roadmap.md` (to be added) for the plan.

## Project structure

```
├── data/
│   ├── raw/              # put the downloaded dataset here (see below)
│   └── processed/        # generated by prepare_dataset.py — train/val/test splits

│   ├── models/
│   │   └── architectures.py     # CNNBaseline, ResNet50, EfficientNet-B0

│   │   └── utils.py
│   └── evaluation/
│       └── evaluate.py          # accuracy/precision/recall/F1/ROC-AUC/confusion matrix

│   ├── logs/              # per-model training history (JSON)

│   └── heatmaps/          # reserved for Module 3 (Grad-CAM)

```

## Setup

```bash
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

GPU is strongly recommended for ResNet50/EfficientNet training but not
required — `train.py` falls back to CPU automatically.

## Dataset

This repo does not ship data. Download the **IQ-OTH/NCCD Lung Cancer
Dataset** yourself (search Kaggle for "IQ-OTH/NCCD lung cancer dataset") and
arrange it like this before running anything:

```
data/raw/
├── Normal/
│   ├── image1.jpg
│   └── ...
├── Benign/
│   └── ...
└── Malignant/
    └── ...
```

Then build the train/val/test split (70/15/15, stratified per class):

```bash
python -m src.data.prepare_dataset
```

## Training

Train one architecture:

```bash
python -m src.training.train --model resnet50
```

Train all three back-to-back for the comparison table required by the spec:

```bash
python -m src.training.train --model all
```

Useful flags: `--epochs`, `--lr`, `--batch-size`. Defaults live in `src/config.py`.

Each run saves the best checkpoint (by validation accuracy) to
`checkpoints/<model_name>_best.pt` and training history to
`outputs/logs/<model_name>_history.json`.

## Evaluation

```bash
python -m src.evaluation.evaluate --model all
```

Writes, per model, to `outputs/metrics/`:
- `<model>_classification_report.txt` — precision/recall/F1 per class
- `<model>_confusion_matrix.png`
- `<model>_roc_curves.png` — one-vs-rest ROC per class
- `<model>_metrics.json` — accuracy, weighted precision/recall/F1, macro ROC-AUC

## Notes on design choices (useful for your viva)

- **Class balancing**: IQ-OTH/NCCD is not evenly split across the three
  classes, so training uses a `WeightedRandomSampler` (oversamples minority
  classes) rather than naive shuffling.
- **CNN baseline is trained from scratch on purpose** — it's there to give
  you a real number to point to when you argue transfer learning helped,
  rather than assuming it.
- **Grad-CAM target layers are pre-wired** (`get_target_layer` in
  `architectures.py`) for all three backbones so Module 3 can plug straight
  in without touching model code again.
- **Early stopping + `ReduceLROnPlateau`** both watch validation loss;
  checkpointing separately watches validation accuracy — so the saved model
  is the accuracy-best one, not just the last one before stopping.
